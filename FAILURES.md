# FAILURES.md — permanent production failure registry

> Every entry here is injected into ALL topic/script prompts as a hard rule
> (see `script_gen.load_learnings`). When a review finds a new failure class,
> add ONE short entry: what went wrong → the rule that prevents it.
> Keep entries general (failure *classes*, not one-off notes).

1. **Scuba diver in an unprotected-human premise** (Mariana long-form).
   Rule: the scenario's protagonist must never be depicted by stock footage
   of equipped/different humans — hero image carries protagonist beats;
   forbidden_visuals must list premise-breaking equipment.

2. **Mixed hypothetical scenarios** (oxygen short: atmospheric O₂ loss shown
   with all-oxygen-atoms consequences like "concrete turns to dust").
   Rule: one interpretation chosen in scene 1; every consequence derives from
   it; state the boundary when it sharpens the hook.

3. **Overstated physics** (Mariana: "pressure flattens the whole body").
   Rule: bodies are mostly water; pressure catastrophically affects GAS-FILLED
   spaces first (ears, sinuses, lungs). Name the mechanism precisely; use
   "अनुमान/लगभग" hedging when simplifying.

4. **Metaphorical/comical stock for abstract mechanisms** (coffee poured on
   ice for "unavailable water"; gauge that resembled a toilet).
   Rule: abstract mechanisms get diagrams/graphics (stat/card/glass/map),
   never literal-object metaphors from stock libraries.

5. **Static graphic dominating the frame for 10+ seconds** (कंक्रीट = धूल card).
   Rule: overlay graphics are impact moments (~3-5s), footage carries scenes;
   never let a text card be the only visual state for a long stretch.

6. **Undelivered runtime** (6:00 target rendered 4:38; 28s short rendered 38s).
   Rule: respect the HARD RANGE word budgets; the word count IS the runtime.
